# Admin Operations — Restaurant Catering Follow-Up Copilot

## Daily operating checklist

1. Review failed imports and mapping errors.
2. Check overdue high-value follow-up tasks.
3. Sample message drafts for menu/pricing accuracy, opt-out handling, and tone.
4. Review suppression changes and bounced/complaint flags.
5. Update pilot tracker with follow-ups completed and revenue attributed.

## Weekly operating checklist

- Produce pilot progress summary for each restaurant.
- Compare recovered/influenced revenue to pilot fee.
- Identify accounts with high value but no next touch scheduled.
- Review template performance by segment: quote, first order, repeat, lapsed, seasonal.
- Confirm menu links, delivery caveats, and minimum order notes are still current.

## Human review rules

- No automatic outreach in MVP; staff must approve/copy/export drafts.
- Do not invent prices, discounts, delivery availability, or menu items.
- Allergy/dietary language must be factual and staff-reviewed.
- Complaints/refunds are routed to staff, not AI-generated resolution.
- Suppressed contacts are excluded from queues and exports by default.

## Customer onboarding

1. Create restaurant workspace and locations.
2. Capture catering menu URL, minimums, delivery caveats, brand tone, and staff users.
3. Import sanitized last-90-days catering leads/orders or manually enter sample data.
4. Run duplicate/account matching review.
5. Generate first prioritized follow-up list.
6. Review 10 sample drafts with owner/catering manager.
7. Start 30-day pilot reporting cadence.

## Pilot reporting template

- Imported contacts/orders.
- Follow-up tasks created.
- Follow-ups completed.
- Orders recovered or influenced.
- Estimated recovered revenue.
- Staff time required.
- Objections or workflow friction.
- Recommendation: continue monthly, adjust segment, or disqualify.

## Support playbooks

- **Bad import**: quarantine batch, correct mapping, reprocess; never silently drop rows.
- **Duplicate accounts**: merge only when confident; preserve original records in audit trail.
- **Wrong menu/pricing mention**: mark draft unsafe, update menu rules, regenerate affected drafts.
- **Opt-out complaint**: suppress contact immediately and inspect audit/export history.
