# Repo Description — msp-security-reporting

## Product

**MSP Security Reporting Copilot**

A validation prototype for MSP owners who need monthly security reporting that explains what was protected, what improved and what needs client approval next.

## Purpose inside `output/businesses/`

This folder is an incubated business workspace inside the main GitHub repo. It is not a separate remote repo yet. It should contain the documentation, frontend, backend, admin, public-content, design context, and validation assets needed to evaluate whether the business deserves its own dedicated repo later.

## Target buyer / user

Buyer segment to be validated.

## Product promise

Turn quiet MSP security work into client-ready proof-of-value reports.

## Current status

Honest label: **draft/incubation**. This is a validation workspace, not production-ready software.

## Key files/folders

- `DESIGN.md` — design tokens and brand system.
- `IMAGE_REQUIREMENTS.md` — image backlog and specs.
- `contextoparalaimagenconcepto.md` — prompt/context bridge for concept image generation.
- `frontend/` — public-facing validation UI.
- `admin/` — operator/admin panel layer.
- `backend/` — API/service layer.
- `docs/` — product/business/technical docs.
- `public-content/` — build-in-public and outreach assets.
